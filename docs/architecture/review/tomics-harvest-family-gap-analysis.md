# TOMICS Harvest Family Gap Analysis

## Exact-source paths completed

These completed items refer to traced subrules and committed interfaces, not to a blanket claim that every research family already runs as a native end-to-end literature runtime.

- TOMSIM truss readiness and whole-truss harvest
- TOMSIM linked-truss leaf harvest threshold handling
- De Koning `FDS >= threshold` harvest readiness
- De Koning vegetative-unit / first-fruit-colour leaf harvest rule
- De Koning FDMC functions in `fds` and harvest-time modes
- Vanthoor fixed boxcar last-stage harvest semantics
- Vanthoor max-LAI pruning flow / explicit leaf-harvest seam
- Kuijpers `h1` / `h2` scaffold mapping
- the external-harvest zero-yield replay seam is closed and post-writeback dropped-mass audit is now explicit

## Research-proxy seams still explicit

- TOMGRO harvest outflow remains a mature-class / mature-pool proxy because the public runtime still does not carry a native TOMGRO age-class mature-pool implementation
- TOMGRO leaf harvest remains management-linked rather than a fully exact-source pruning rule
- De Koning runtime integration remains adapter-based around the traced readiness and FDMC subrules, and fruit-position weighting remains a documented research seam rather than an exact-source committed production law
- Vanthoor runtime integration remains adapter-based, and the stem-root back-mapping is a reporting map rather than a claimed literature stem/root split

## What still remains deferred

- family-specific calibration beyond the current reduced KNU harvest screening
- richer fruit-position weighting inside De Koning harvest events
- exact measured EC / root-zone temperature override data for the De Koning harvest-time FDMC path
- measured greenhouse pruning-management logs to replace current harvest-delay / leaf-removal management assumptions
- native Vanthoor fixed-boxcar runtime
- native TOMGRO mature-pool runtime
- family-specific explicit leaf-harvest runtimes beyond the current management-linked proxy paths

## Decision

The current repository now has a literature-aware harvest family layer and a repaired harvest replay path, but the promotion gate still does not justify changing the public incumbent baseline.
