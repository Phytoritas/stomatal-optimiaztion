# Canonical I/O

## Inputs
- forcing data
- parameter tables
- configuration files
- canonical variable names and units
- optional calibration targets

## Outputs
- canonical state trajectories
- canonical flux trajectories
- summary metrics
- representative plots or tables
- validation artifacts

## Artifact policy
- Keep representative outputs only.
- Keep repeated run artifacts out of Git.
- Write generated outputs to `artifacts/`, `results/`, or another ignored path.
